        <ContextMenu.Item>Name Window…</ContextMenu.Item>
        <ContextMenu.Separator />
        <ContextMenu.Item>Developer Tools</ContextMenu.Item>
      </ContextMenu.SubContent>
    </ContextMenu.Sub>

    <ContextMenu.Separator />

    <ContextMenu.CheckboxItem shortcut="⌘ B" checked>
      Show Bookmarks
    </ContextMenu.CheckboxItem>
    <ContextMenu.CheckboxItem>Show Full URLs</ContextMenuCheckboxItem>

    <ContextMenu.Separator />

    <ContextMenu.Label>People</ContextMenu.Label>
    <ContextMenu.RadioGroup value="pedro">
      <ContextMenu.RadioItem value="pedro">Pedro Duarte</ContextMenu.RadioItem>
      <ContextMenu.RadioItem value="colm">Colm Tuite</ContextMenu.RadioItem>

## API Reference

This component inherits props from the [Radix ContextMenu](/primitives/docs/components/context-menu) primitive.

### Root

Contains all the parts of a context menu.

### Trigger

Wraps the element that will open the context menu.

### Content

The component that pops out when the context menu is open.

<ThemesPropsTable name="contextMenuContent" />

### Label

Used to render a label. It won't be focusable using arrow keys.

### Item

The component that contains the context menu items.

<ThemesPropsTable name="contextMenuItem" />

### Group

Used to group multiple `ContextMenu.Item`s.

### RadioGroup

Used to group multiple `ContextMenu.RadioItem`s.

### RadioItem

An item that can be controlled and rendered like a radio.

### CheckboxItem

An item that can be controlled and rendered like a checkbox.

### Sub

Contains all the parts of a submenu.

### SubTrigger

An item that opens a submenu. Must be rendered inside `ContextMenu.Sub`.

### SubContent

The component that pops out when a submenu is open. Must be rendered inside `ContextMenu.Sub`.

## Examples

### Size

Use the `size` prop to control the size.

```jsx live=true
<Grid columns="2" gap="3">
  <ContextMenu.Root>
    <ContextMenu.Trigger>
      <RightClickZone />
    </ContextMenu.Trigger>
    <ContextMenu.Content size="1">
      <ContextMenu.Item>New Tab</ContextMenu.Item>
      <ContextMenu.Item>New Window</ContextMenu.Item>
    </ContextMenu.Content>
  </ContextMenu.Root>

  <ContextMenu.Root>
    <ContextMenu.Trigger>
      <RightClickZone />
    </ContextMenu.Trigger>
    <ContextMenu.Content size="2">
      <ContextMenu.Item>New Tab</ContextMenu.Item>
      <ContextMenu.Item>New Window</ContextMenu.Item>
    </ContextMenu.Content>
  </ContextMenu.Root>
</Grid>
```

### Variant

Use the `variant` prop to customize the visual style of the context menu.

```jsx live=true
<Grid columns="2" gap="3">
  <ContextMenu.Root>
    <ContextMenu.Trigger>
      <RightClickZone />
    </ContextMenu.Trigger>
    <ContextMenu.Content variant="solid">
      <ContextMenu.Item>New Tab</ContextMenu.Item>
      <ContextMenu.Item>New Window</ContextMenu.Item>
    </ContextMenu.Content>
  </ContextMenu.Root>

  <ContextMenu.Root>
    <ContextMenu.Trigger>
      <RightClickZone />
    </ContextMenu.Trigger>
    <ContextMenu.Content variant="soft">
      <ContextMenu.Item>New Tab</ContextMenu.Item>
      <ContextMenu.Item>New Window</ContextMenu.Item>
    </ContextMenu.Content>
  </ContextMenu.Root>
</Grid>
```

### Color

Use the `color` prop to assign a specific [color](/themes/docs/theme/color), ignoring the global theme. You can also pass `color` to a specific item to override for semantic reasons (ie. destruction action).
